# Play Store Screenshot Runbook

This is the practical capture guide for Home Basket's first Android store listing.

Use this together with:

- [Play Store creative brief](./play-store-creative-brief.md)
- [Store listing copy](./store-listing-copy.md)
- [Release QA checklist](./release-qa-checklist.md)

## Goal

Capture a polished screenshot set from the real app UI that tells a clear MVP story.

## Capture format

Recommended export target:

- `1080 x 1920`
- PNG
- portrait

If you capture from a higher-resolution phone, export down to a clean final size afterward.

## Visual rules

- Use one sample household across the whole set.
- Keep the same member names throughout.
- Hide any developer-only or temporary debugging elements.
- Use tidy, intentional example data.
- Keep one consistent currency across the set.
- Do not use real personal receipt details unless sensitive fields are hidden.

## Capture workflow

1. Open the app on a real Android device if possible.
2. Use a clean sample household with curated data.
3. Capture full screenshots with no system clutter if possible.
4. Select the best screenshot for each story step.
5. Add the store headline as a clean overlay in Canva, Figma, or similar.
6. Export final store assets into `branding/play-store/`.

## Overlay guidance

- Use large bold type only
- Keep to 1 short headline per screenshot
- Avoid paragraphs
- Use the Home Basket green/gold palette
- Do not cover the strongest UI content

## Final check before upload

- text is readable at thumbnail size
- branding is consistent
- no private data is visible
- screenshots feel like one product family
- first 3 screenshots tell the story even without reading the description
